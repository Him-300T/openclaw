# 提交模板
## 标准提交信息格式

## 摘要格式
```
类型: 简要描述
```

### 类型选项
- `feat` - 新功能/内容
- `fix` - 修复问题  
- `docs` - 文档更新
- `style` - 格式调整（不影响功能）
- `refactor` - 代码重构
- `test` - 测试相关
- `chore` - 维护任务

### 示例摘要
```
feat: 添加OpenClaw学习笔记
fix: 修复记忆同步脚本路径问题
docs: 更新项目README文档
```

## 详细描述模板

### 新增内容
```
## 新增内容
- 功能1：简要描述
- 功能2：简要描述
- 文档：文档名称和用途
```

### 修改内容  
```
## 修改内容
- 文件1：修改原因和内容
- 文件2：优化和改进点
- 配置：配置变更说明
```

### 技术价值
```
## 技术价值
- 解决的问题：描述解决的具体问题
- 带来的改进：性能、可用性等方面的改进
- 学习收获：技术学习和经验总结
```

### 备注
```
## 备注
- 特殊说明：需要特别注意的事项
- 后续计划：下一步工作计划
- 依赖关系：与其他功能/文件的关联
```

## 提交频率建议

### 小步提交
```
优点：
- 版本清晰，易于理解
- 回滚方便，风险低
- 问题定位准确

示例：
feat: 添加文章笔记模板
feat: 实现文章内容解析
feat: 添加文章索引功能
```

### 功能完整提交
```
优点：
- 提交记录整洁
- 功能完整性好
- 适合功能模块

示例：
feat: 完成文章学习系统

## 新增内容
- 文章解析模块
- 笔记生成系统
- 索引管理功能
- 导出工具

## 技术价值
- 完整的工作流支持
- 模块化设计，易于扩展
- 良好的用户体验
```

## 特殊提交说明

### 合并提交
```
当需要合并多个相关小提交时：

feat: 合并近期学习成果更新

## 包含内容
- 文章笔记更新（3篇）
- 学习计划调整
- 文档格式优化

## 备注
- 合并了2026-04-17至2026-04-20的提交
- 保持功能完整性
- 清理提交历史
```

### 重构提交
```
refactor: 重构文件同步架构

## 修改内容
- 架构：从单体脚本改为模块化设计
- 接口：定义清晰的模块接口
- 测试：增加单元测试覆盖

## 技术价值
- 可维护性：模块化设计便于维护
- 可测试性：独立的模块易于测试
- 可扩展性：方便添加新功能

## 备注
- 功能不变，仅架构调整
- 保持向后兼容
- 性能略有提升
```

## 使用建议

### 提交前检查
1. **功能完整**：确保提交的功能完整可用
2. **测试通过**：相关功能已测试验证
3. **文档更新**：相应文档已同步更新
4. **代码规范**：符合项目代码规范

### 提交后验证
1. **构建成功**：确认提交后项目可正常构建
2. **功能正常**：验证相关功能正常工作
3. **文档一致**：检查文档与实际功能一致

## 模板使用
复制相应模板，根据实际情况修改内容，保持描述清晰准确。
