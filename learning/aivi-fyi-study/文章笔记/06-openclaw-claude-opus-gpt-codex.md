## Claude Opus 4.6：超长上下文与稳定性

### 定位
"超大材料输入 + 高可靠总结与推理"的强化版

### 核心升级
1. **百万级别上下文窗口**（测试阶段）
2. **最大输出提升到128K tokens**
3. **长上下文下的"抗衰减"能力**
4. **分段计费策略**

### 适用场景
- 跨多个文档的合同审阅
- 产品资料与需求文档合并
- 年度经营数据与会议纪要综合分析
- 长篇研究报告的结构化改写

### 关键优势
1. **减少拆分**：避免将长文档拆成几十段
2. **减少重复提示**：保持上下文一致性
3. **减少人为误差**：自动化处理减少中间环节
4. **稳定输出**：在超长材料中仍能定位关键细节

### 成本策略
- 常规上下文范围内较低档计费
- 超过超长阈值后进入更高档计费
- 鼓励将"长上下文"用在真正需要的地方

## GPT-5.3-Codex：从代码生成转向可指挥的执行型Agent

### 定位
"可以被指挥完成任务"的执行型智能体

### 核心升级
1. **更强的agentic coding能力**
2. **理解仓库结构、分析报错日志**
3. **在多轮迭代中持续修复问题**
4. **按约束提交可用补丁**
5. **长任务过程中允许人随时插话纠偏**

### 适用场景
- 仓库级别的修改
- 修复bug、补测试
- 跑命令、看日志
- 迭代补丁直到通过验收

### 关键优势
1. **执行与协作**：更像在带一个能动手的同事
2. **工程落地导向**：面向真实工程任务的评测表现
3. **大上下文容量**：总上下文400K tokens，输出128K tokens
4. **成本优势**：输入输出单价控制在易规模化使用区间

### 技术规格
- **总上下文**：400K tokens
- **最大输出**：128K tokens
- **缓存输入机制**：降低反复迭代同一份上下文的成本

### 安全与执行边界
- 隔离环境中执行
- 默认限制网络
- 对潜在风险操作设定约束
- 决定能否被更大范围团队接入真实工作流的前提

- **价值**：测试模型的逻辑推理和优化能力

### 2. 农夫过河问题
- **问题**：农夫带老虎、羊、蛇、鸡、苹果过河
- **约束**：复杂的相互制约关系
- **价值**：测试模型的约束满足和规划能力

### 3. UI复刻
- **要求**：用最适合的前端技术复刻图中的UI
- **价值**：测试模型的技术理解和实现能力

### 4. SVG生成
- **要求**：用SVG画出Agent teams架构图，具备动态效果
- **价值**：测试模型的架构理解和可视化能力

### 5. 冒泡排序动画
- **要求**：用Python+Pygame创建冒泡排序动画
- **价值**：测试模型的算法理解和编程能力

### 6. 审查Bug测试

- **价值**：测试模型的代码分析和问题诊断能力
